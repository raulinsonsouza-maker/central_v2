rd station como passar utm correta de camapanha criativo para cadastrar no rd

Para o RD Station Marketing captar corretamente as UTMs da campanha, conjunto e criativo (Meta Ads ou Google Ads), você precisa garantir 2 coisas:

Os parâmetros UTM na URL do anúncio

O formulário/landing page integrado corretamente ao RD

Padrão recomendado de UTM
Meta Ads
Use este modelo na URL do anúncio:

https://seudominio.com.br/pagina?utm_source=facebook&utm_medium=paid_social&utm_campaign={{campaign.name}}&utm_content={{ad.name}}&utm_term={{adset.name}}
O que cada campo salva no RD
UTM	O que vai aparecer
utm_source	facebook
utm_medium	paid_social
utm_campaign	nome da campanha
utm_content	nome do criativo/anúncio
utm_term	nome do conjunto
Tokens dinâmicos da Meta
A Meta for Developers permite usar parâmetros dinâmicos:

Token	Retorna
{{campaign.name}}	Nome da campanha
{{adset.name}}	Nome do conjunto
{{ad.name}}	Nome do anúncio/criativo
{{placement}}	Posicionamento
{{site_source_name}}	facebook / instagram
Modelo mais profissional (recomendado)
https://seudominio.com.br/?utm_source={{site_source_name}}&utm_medium=paid_social&utm_campaign={{campaign.name}}&utm_term={{adset.name}}&utm_content={{ad.name}}
Assim o RD vai registrar:

facebook ou instagram

campanha

conjunto

criativo exato

?utm_source=google&utm_medium=cpc&utm_campaign={campaignid}&utm_content={creative}&utm_term={keyword}
Ou usando nomes:

?utm_source=google&utm_medium=cpc&utm_campaign={campaignname}&utm_content={creative}&utm_term={keyword}

Sua IA não “puxa leads sozinha” — ela depende de:

✅ Page Access Token (já vimos que você tem)
Precisa ser token da PÁGINA

Não pode ser user token

✔ Você já resolveu isso

⚠️ Mas precisa garantir também:
✔ Permissões ativas no token:
leads_retrieval (principal)

pages_show_list

pages_read_engagement

⚠️ E o mais importante:
👉 A página precisa estar LIBERADA no:

Meta Business Suite → Lead Access Manager

Se isso não estiver liberado:

API retorna vazio

webhook não dispara

leads “não existem” na API

2. 📡 VOCÊ PRECISA DECIDIR O MÉTODO DE CAPTURA
Aqui é onde sua IA “nasce de verdade”.

Você tem 2 formas:

🥇 OPÇÃO RECOMENDADA: WEBHOOK (TEMPO REAL)
📌 Fluxo correto:
Meta Graph API envia evento →

seu backend recebe →

sua IA processa →

seu dashboard atualiza

🔥 O QUE VOCÊ PRECISA CONFIGURAR:
1. Webhook no app Meta
No seu app:

Products → Webhooks

Seleciona: Page

Evento: leadgen

2. Endpoint no seu servidor
Exemplo:

https://seu-dominio.com/webhook/meta
Esse endpoint precisa:

receber POST da Meta

responder 200 rápido

armazenar lead_id

3. Verificação (obrigatório)
Meta vai chamar:

GET ?hub.challenge=
Sua API precisa responder isso corretamente.

🥈 OPÇÃO 2: POLLING (menos eficiente)
Se não quiser webhook:

Sua IA precisa rodar rotina:

GET /FORM_ID/leads
ou:

GET /PAGE_ID/leadgen_forms
→ depois /FORM_ID/leads
⛔ Problema:

atraso

risco de perder lead

mais custo de API

3. 🧠 SUA IA PRECISA FAZER “ENRIQUECIMENTO DE DADOS”
Quando o lead chega, ele NÃO vem completo.

Ele só traz IDs:

campaign_id

adset_id

ad_id

Então sua IA precisa SEMPRE rodar isso:
📌 1. Buscar nome da campanha
/{campaign_id}?fields=name
📌 2. Buscar conjunto
/{adset_id}?fields=name
📌 3. Buscar criativo
/{ad_id}?fields=name,creative
🔥 Isso é o que transforma “dado bruto” em inteligência
Sem isso:
→ seu dashboard só mostra números

Com isso:
→ você vê criativo, público e campanha real

4. 🧱 ARQUITETURA QUE SUA IA PRECISA TER
Aqui está o ponto mais importante:

📊 PIPELINE REAL
Meta Lead Ads
   ↓
Webhook (ou polling)
   ↓
Validação do lead
   ↓
Enrichment (campaign/ad/adset)
   ↓
Normalização do lead
   ↓
Envio para banco
   ↓
Dashboard
5. 🗄️ SUA IA PRECISA SALVAR ISSO (MÍNIMO)
Se não salvar isso, não existe dashboard útil:

📌 Lead base:
lead_id

created_time

form_id

📌 Dados do usuário:
nome

email

telefone

📌 Marketing data:
campaign_id + name

adset_id + name

ad_id + name

📌 Origem:
facebook / instagram

placement (se disponível)

6. ⚠️ ERRO QUE 90% ERRAM
Você NÃO pode depender só de:

/leadgen_forms
Esse endpoint:

❌ NÃO traz leads
❌ só lista formulários

Você precisa SEMPRE usar:

/{form_id}/leads
OU webhook leadgen
7. 🔥 O QUE SUA IA PRECISA “DECIDIR”
Ela precisa ter lógica:

📌 Regra principal:
“Lead só entra no dashboard se vier com campaign_id + ad_id válidos”

📌 Regra de segurança:
duplicado? ignora

sem campaign_id? marca como “orgânico inválido”

sem ad_id? não atribui criativo

🚀 RESUMO FINAL (O ESSENCIAL)
Sua IA precisa ter 4 coisas:

1. Conexão correta
Page Access Token

leads_retrieval ativo

página liberada no Meta Business

2. Método de captura
webhook (ideal)
OU

polling via /form_id/leads

3. Enriquecimento obrigatório
campaign_id → name

adset_id → name

ad_id → name + creative

4. Pipeline interno
receber lead

normalizar dados

salvar no banco

enviar para dashboard

arquitetura pronta (Node + webhook + DB)

schema do banco ideal pro seu dashboard

fluxo completo tipo SaaS de CRM

ou integração direta com RD Station + Meta ao mesmo tempo

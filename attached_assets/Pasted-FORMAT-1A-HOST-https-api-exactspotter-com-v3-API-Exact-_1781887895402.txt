API - Exact Spotter V2
O objetivo deste documento é auxiliar os clientes da Exact Sales na execução de atividades oferecidas
no sistema do Exact Spotter.
A API do Exact Spotter disponibiliza um serviço para acessar e manipular dados sob o protocolo HTTP
no padrão OData. Este protocolo permite que você realize filtros e ordenações utilizando "comandos"
como select, filter, top, count, etc. Para mais informações, recomendamos este tutorial.
Por padrão os registros serão retornados de 500 em 500, mas por meio do método $skip é possível
fazer a paginação dos registros retornados.
URL para as requisições: https://api.exactspotter.com/v3
Autenticação: em todas as chamadas deverá ser informado no header o parâmetro com token da
empresa (token_exact = token). O token deverá ser gerado no sistema Exact Spotter, em
CONFIGURAÇÕES > INTEGRAÇÕES. Importante: Não compartilhe o token!
Rate limit: todos os endpoints têm um limite de 30 requisições para cada intervalo de 20 segundos.
Todos os campos data/hora estão no fuso horário UTC.
Os dados enviados devem obedecer o padrão UTF-8.
Quando o parâmetro estiver entre { }, o mesmo deverá ser enviado sem elas.
ATENÇÃO ao ajuste nos endpoints abaixo da internacionalização dos telefones DDI + DDD sem o 0 e
o número.

phone2: Segunda opção do DDD + Telefone da empresa/organização sem o 0;
website = Referente ao site da empresa/organização;
address = Endereço da empresa/organização;

city = A cidade que a empresa/organização se encontra;
state = O estado desta;
country = País que a empresa/organização se localiza;
description = Qualquer observação que se deseja fazer a respeito da empresa/organização que
está sendo inserida;
customFields = Permite atualizar ou inserir um campo personalizado em uma
empresa/organização.

Para "options" o "id" será o id da opção.
Campo obrigatório:
id = identificador da empresa/organização;

zipcode então city, state e country são obrigatórios.

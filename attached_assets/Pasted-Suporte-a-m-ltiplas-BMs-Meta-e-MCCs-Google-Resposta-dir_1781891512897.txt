Suporte a múltiplas BMs (Meta) e MCCs (Google)

Resposta direta à sua dúvida

Hoje, não — o sistema não suporta múltiplos tokens Meta nem múltiplos logins OAuth Google em paralelo.







Plataforma

Meta



1 token global + ID da conta de anúncio por cliente (Conta.accountIdPlataforma)



2+ tokens/BMs diferentes (Inout + Mata Sede + Bruno Reis)





Google



1 OAuth global + customer ID por cliente + MCC opcional por cliente (Conta.googleAdsLoginCustomerId)



MCCs de contas Google diferentes com refresh tokens distintos

O resultado final para o cliente (dashboard, sync, leads) já é por cliente — o que falta é escolher qual credencial usar em cada sync.

Caminho recomendado (você confirmou tokens separados)

Introduzir o conceito de Conexão de Integração — um perfil reutilizável por BM/MCC, ao qual vários clientes podem apontar.

  subgraph conexoes [Conexões cadastradas]

    G2[Bruno Reis MCC - Google]
  end

  ClienteA[Cliente Inout] -->|Conta META| C1
  ClienteB[Cliente Bruno Reis] -->|Conta META| C2
  ClienteC[Cliente Mata Sede] -->|Conta META| C3

  Sync[Sync diário] -->|resolve credencial da Conta| conexoes
  Sync --> Dashboard[Dashboard do cliente - inalterado]

1. Novo modelo Prisma: ConexaoIntegracao

model ConexaoIntegracao {
  id        String   @id @default(cuid())
  nome      String   // ex: "Inout Principal", "Bruno Reis BM"
  plataforma String  // "META" | "GOOGLE_ADS"
  ativo     Boolean  @default(true)

  // Meta
  metaAccessToken String?

  // Google Ads (conjunto OAuth completo por conexão)
  googleClientId        String?
  googleClientSecret    String?
  googleDeveloperToken  String?
  googleRefreshToken    String?
  googleLoginCustomerId String?

  contas    Conta[]
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

Alterar Conta:

conexaoIntegracaoId String?
conexaoIntegracao   ConexaoIntegracao? @relation(...)
